# Nook 웹 프론트엔드 디자인 기록

## 디자인 방향

**“따뜻하고 편안한 느낌을 유지하고 미니멀리스틱하게 디자인한다.”**

Nook는 흩어진 영감을 수집하고 여러 편의 원고로 발전시키는 개인 글쓰기 공간이다. Brunch의 넓은 여백과 콘텐츠 중심 구성을 참고하되, 세이지 그린·크림색 팔레트와 ‘조용한 작업실’의 공간감으로 독립적인 인상을 만들었다. 서비스명은 모든 화면에서 `Nook`로 통일했다.

## 시각 시스템

- 배경은 크림색(`#F8F6F1`), 콘텐츠 면은 종이색(`#FFFEFA`)이다.
- 주요 동작은 낮은 채도의 세이지 그린(`#3F4740`)으로 표시한다.
- 제목과 원고는 `Gowun Batang`, 기능 UI는 `Noto Sans KR`을 사용한다.
- 그림자를 절제하고 1px 테두리와 여백으로 정보 층위를 만든다.

## 화면 구조

| 경로                                           | 역할                                     |
| ---------------------------------------------- | ---------------------------------------- |
| `/login`                                       | Google 로그인과 서비스 소개              |
| `/`                                            | 빠른 기록, 최근 글감, 진행 프로젝트      |
| `/captures`, `/captures/new`, `/captures/[id]` | 글감 목록·등록·상세                      |
| `/projects`, `/projects/new`                   | 프로젝트 목록·생성                       |
| `/projects/[id]`                               | 프로젝트 개요, 복수 원고 목록, 연결 글감 |
| `/projects/[id]/write/[manuscriptId]`          | 선택 원고 편집과 글감 삽입 패널          |
| `/profile`                                     | 사용자 정보, 통계, 계정 설정             |

## 핵심 UX 결정

- 사이드바에서 독립 ‘글쓰기’를 제거했다. 프로젝트 안의 원고를 클릭해야 편집기로 이동한다.
- 한 프로젝트의 원고를 번호·제목·요약·글자 수·수정일로 비교할 수 있다.
- 편집기는 본문에 집중하는 전용 화면이며 오른쪽 연결 글감을 삽입할 수 있다.
- 사이드바는 태블릿에서 아이콘형, 모바일에서 슬라이드 메뉴로 전환한다.

## 백엔드 연결 경계

현재 데이터는 `apps/web/app/data.ts`의 더미 데이터다. 코드의 `TODO(backend)` 주석은 Auth/프로필, captures CRUD·Storage, projects/documents CRUD, project_captures, 편집기 자동 저장 연결 지점을 표시한다.

## 버전 기록

- `v0.1` — Nook 명칭, 전체 MVP 화면, 복수 원고 흐름, 반응형 UI, 더미 인터랙션 구현
- `v0.2` — 프로필 편집·회원 탈퇴, 다중 태그와 태그 관리, 프로젝트 편집·삭제 메뉴, 서비스 설정 화면, 웹·모바일 글감 알림, 상단 계정 팝업 추가

## 개발 하네스

다음 변경에서도 지켜야 하는 구조 및 검증 규칙이다.

### 코드 구조

- 페이지는 라우팅과 데이터 선택을 담당하고, 상태가 있는 큰 UI는 기능별 Client Component로 분리한다.
- 공통 앱 프레임은 `app/components/app-shell.tsx`, 공통 표시 컴포넌트는 `app/components.tsx`에서 관리한다.
- 더미 타입과 데이터는 `app/data.ts`에 모으고 모든 더미 데이터 블록에 `TODO(backend)`를 유지한다.
- 반복되는 화면 데이터는 배열과 명시적 TypeScript 타입으로 모델링하며 중첩 삼항 연산자를 사용하지 않는다.
- JSX 속성과 이벤트가 길어지면 여러 줄로 나누고, 이름은 역할과 행동이 드러나게 작성한다.

### 기능 불변 조건

- 좌측 내비게이션에는 홈·글감함·프로젝트·설정만 둔다.
- 글쓰기는 반드시 프로젝트의 특정 원고를 통해 진입한다.
- 한 글감은 0개 이상의 태그를 가질 수 있고, 태그 삭제 시 연결 관계 처리 여부를 사용자에게 알려야 한다.
- 프로젝트 삭제와 회원 탈퇴처럼 되돌릴 수 없는 동작에는 확인 단계를 둔다.
- 설정은 `/settings`, 개인 정보와 계정 관리는 `/profile`에서 담당한다.
- 알림에는 글감이 추가된 환경(모바일/웹), 제목, 시간, 읽음 상태가 포함되어야 한다.
- 팝업은 다시 누르거나 바깥 영역을 누르면 닫혀야 하며 모바일 화면 밖으로 넘치지 않아야 한다.

### 완료 전 검증

1. `npm run build`로 TypeScript와 전체 라우트 빌드를 통과한다.
2. 데스크톱 1280px, 태블릿 900px, 모바일 390px에서 주요 화면의 넘침을 확인한다.
3. 프로필 수정, 태그 다중 선택·생성·삭제·필터, 프로젝트 수정·삭제 확인, 설정 저장을 조작한다.
4. 알림과 계정 팝업의 열기·닫기·외부 클릭 동작을 확인한다.
5. `Brish` 잔존 문자열과 `TODO(backend)` 누락 여부를 검색한다.
